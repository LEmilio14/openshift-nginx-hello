Introduction to OpenShift Container Platform

Containers and Kubernetes can help ease the burden of configuring, deploying, managing, and monitoring even the largest-scale applications. Together, they provide organizations with the ability to develop applications once and deploy them anywhere - regardless of the location. Developers can release code without worrying about the underlying hardware infrastructure, while the operator framework provides automation solutions needed to simplify day 1 and day 2 operations. Thus, IT Ops & DevOps teams are able to gain significant advantages and efficiencies through a consistent and scalable platform that runs across on-premises; public, hybrid, or multi clouds; as well as the edge.

This website is powered by OpenShift, which is an Enterprise-Ready Kubernetes solution that is running on just three X12 BigTwin nodes. These nodes consist of 3rd Gen Intel® Xeon® Scalable Processors, Intel® Optane™ Persistent Memory 200, Intel® Optane™ DC SSD Series, and Intel Ethernet 800 Series. The 4th node of this platform supports 2 other key solutions: Bastion and SuperCloud Composer. This compact cluster makes it easy to deploy and manage a hybrid cloud within only 2U of rack space.

The Bastion host serves as the main deployment and management server for the OpenShift Cluster, while SuperCloud Composer runs in a VM as a supplemental composable cloud management platform that provides a unified dashboard to administer software-defined data centers. Adding additional storage (such as Ceph storage), or even GPU nodes, can be streamlined through SuperCloud Composer and can provide a deeper integration for life cycle management and system analytics. This can include server health, network bandwidth, power metering, as well as external sensors that may be critical to running Data Center operations smoothly.

To learn more about the solution, please refer to our product landing pages and partner's website: X12 BigTwin, X12 Ultra, SuperCloud Composer, Supermicro Solution Integration Center, and Red Hat OpenShift.

Multi-Cloud Object Gateway

  1. Admins can create and manage storage (Back Stores), with the Multi-Cloud Object Gateway
    • Users will be able to create their own Buckets, but cannot create Back Stores from public cloud or on-premise storage.
    Figure 1: OpenShift Multi-Cloud Object Gateway

  2. OpenShift Data Foundation (aka OpenShift Container Storage) will be configured to replicate objects by default
    • Only admins can reconfigurate a Bucket Policy. Replication and Erasure Coding are supported.
    • The on-premise Back Store is supported by the Ceph Cluster running on three X12 Ultra servers with up to 1.1PB of raw storage capacity per 3-node cluster.

How to Access the Cluster

  1. You should already have an established VPN connection to access the OpenShift Console or SuperCloud Composer Dashboard
  2. To login into the OpenShift Console, please click OpenShift Console
  3. To login into the SuperCloud Composer Dashboard, please click SuperCloud Composer Dashboard
    • All users will only have basic user access.
    • For admin controls/requests, please contact _cloudnativesys@supermicro.com.
